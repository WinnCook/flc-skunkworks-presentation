## 4. OPERATING PRINCIPLES

### 4.1 Skunk Works Philosophy
1. **Bias toward action** - Ship early, iterate fast, demonstrate results
2. **Value visibility** - Track and communicate ROI relentlessly
3. **Stakeholder focus** - Every initiative serves a real need
4. **Opportunistic** - Seize emerging opportunities quickly
5. **Documented** - Audit-ready at all times
6. **AI-Augmented** - Leverage AI tools to multiply output

### 4.2 Initiative Selection Criteria
Score each potential initiative (1-5 scale):
| Criterion | Weight | Description |
|-----------|--------|-------------|
| **Impact** | High | How significant is the value delivered? |
| **Visibility** | High | Will stakeholders see and appreciate this? |
| **Feasibility** | Medium | Can this be done with available resources? |
| **Timeliness** | Medium | Is there urgency or a window of opportunity? |
| **Strategic Fit** | High | Does this advance the primary goal? |
| **WOW Factor** | Medium | Does this demonstrate innovation capability? |

**Minimum score to pursue:** 18/30

### 4.3 Quality Standards
- All deliverables professional and leadership-ready
- Data sources documented and verifiable
- Methods transparent and reproducible
- Findings actionable, not just descriptive
- Code/tools documented for handoff or continuation

## 5. GOVERNANCE

### 5.1 Initiative Lifecycle
```
INTAKE → SCORE → CHARTER → EXECUTE → DELIVER → CLOSE
   ↓        ↓        ↓         ↓         ↓        ↓
Identify  Evaluate  Create   Work in   Produce  Archive,
opportunity against  Manifest sessions  artifacts capture
          criteria           & track   & value  lessons
```

### 5.2 Communication Cadence
| Audience | Frequency | Format | Owner |
|----------|-----------|--------|-------|
| VP | Weekly | Brief update + metrics | IR |
| Exec Team | Monthly | Dashboard + highlights | IR via VP |
| Board | Quarterly | Formal presentation | IR via VP |
| Faculty | As needed | Targeted communications | IR |

### 5.3 Session Protocol
This program uses a **stateless workflow** for AI-assisted sessions:
- `continue` - Resume work from Active State
- `SITREP` - Generate status report
- `end session` - Save state, prepare for next session
- All context maintained in files, not conversation memory
